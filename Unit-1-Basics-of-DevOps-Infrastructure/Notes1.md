# Unit 1 – DevOps Basics

## Pre-Knowledge for DevOps

Before learning DevOps, it is important to understand some basic concepts related to software development and system administration.

### 1. Software Development Lifecycle (SDLC)

SDLC is the process used to design, develop, test, and deploy software.
Traditional SDLC stages include:

* Requirement Analysis
* Design
* Development
* Testing
* Deployment
* Maintenance

### 2. Development and Operations

Traditionally, development and operations teams worked separately.

**Development Team**

* Writes and develops the application code.

**Operations Team**

* Manages servers, deployment, and infrastructure.

This separation often caused delays and communication gaps.

### 3. What is DevOps?

DevOps is a combination of **Development (Dev)** and **Operations (Ops)**.

It is a set of practices that aims to:

* Improve collaboration between development and operations teams
* Automate software delivery
* Increase deployment speed
* Improve system reliability

### 4. Goals of DevOps

The main goals of DevOps include:

* Faster software delivery
* Continuous integration and deployment
* Automation of infrastructure
* Better collaboration between teams

### 5. DevOps Tools (Examples)

Common tools used in DevOps include:

* Git – Version control
* Docker – Containerization
* Maven – Build automation
* Jenkins – CI/CD automation
* GitHub Actions – Continuous Integration workflows

These tools help automate the development and deployment process.
